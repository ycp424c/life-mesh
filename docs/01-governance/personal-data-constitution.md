# Personal Data Constitution

状态：draft
最后更新：2026-06-26
职责边界：定义 LifeMesh 处理个人数据时不可绕过的最高约束。

## 必须回答的问题

- 哪些数据永远不进入云端模型？
- 哪些数据只能本地处理？
- 哪些动作必须人工确认？
- 哪些数据可以长期记忆？
- 哪些数据必须定期过期？
- 哪些人际关系、健康、金融、位置或身份信息属于高敏感数据？
- 用户如何查看、修改、导出和删除自己的数据？
- 用户如何撤销 Agent、工具或连接器的授权？

## 初始红线

- 不导入真实敏感样例到仓库。
- 不把聊天全文作为早期默认数据源。
- 不允许 Agent 无审计地读取全量个人数据。
- 不允许高风险动作静默执行。
- 不允许无法解释来源的派生事实被当作确定事实使用。

## 动作风险分级

| 风险级别 | 例子 | 默认策略 |
|---|---|---|
| 低 | 搜索文档、生成摘要、列出待办候选 | 可在授权范围内自动执行 |
| 中 | 写入长期记忆、修改标签、创建提醒草稿 | 需要可见反馈或确认 |
| 高 | 发送消息、提交表单、删除数据、访问金融/健康数据 | 必须人工确认 |
| 极高 | 资金操作、医疗建议、法律承诺、不可逆删除 | 默认禁止，除非有单独授权设计 |
