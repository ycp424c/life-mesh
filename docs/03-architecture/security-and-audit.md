# Security And Audit

状态：draft
最后更新：2026-06-26
职责边界：定义安全、审计、撤销和确认机制的架构要求。

## 必备能力

- 数据分类
- 按任务授权
- 访问审计
- 写入审计
- 授权撤销
- 数据删除
- 记忆纠错
- 高风险动作确认
- 敏感数据隔离

## 审计事件

审计日志至少应覆盖：

- 谁访问了什么数据
- 访问目的是什么
- 使用了哪个 Agent 或工具
- 返回了哪些来源
- 是否产生派生事实或记忆
- 是否执行外部动作
- 用户是否确认、撤销或纠错

## 确认机制

高风险动作默认需要用户确认，包括：

- 发送外部消息
- 修改或删除数据
- 写入长期偏好
- 访问金融、健康、位置、身份数据
- 代表用户做出承诺

## 撤销机制

撤销不应只删除授权记录，还要能追踪：

- 授权期间读取过哪些数据
- 生成过哪些派生事实
- 写入过哪些记忆
- 是否需要清除索引或缓存
