# Provenance And Lifecycle

状态：draft
最后更新：2026-06-26
职责边界：定义数据从原始进入、派生、使用、过期到删除的生命周期。

## 生命周期

1. 发现数据源
2. 完成接入评估
3. 导入 Raw Vault
4. 抽取规范化事实
5. 建立索引、图谱或时间线
6. 授权给 Agent 或工具使用
7. 记录使用和变更审计
8. 用户纠错、撤销或删除
9. 到期归档或清除

## 溯源要求

派生事实必须能回答：

- 来自哪个原始数据？
- 何时抽取？
- 由什么规则、模型或人工操作生成？
- 置信度是多少？
- 最近被哪个 Agent 或工具使用？
- 是否被用户确认或纠正过？
