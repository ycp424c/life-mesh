# Sandboxing And Tool Safety

状态：draft
最后更新：2026-07-03
职责边界：定义 Agent 调用工具时的安全边界和能力分级。

## 工具风险级别

| 级别 | 能力 | 默认策略 |
|---|---|---|
| Readonly | 查询、搜索、摘要 | 授权范围内允许，记录审计 |
| Draft | 创建草稿、建议变更 | 允许生成，写入前确认 |
| Write | 写入记忆、任务、标签 | 需要明确授权和可撤销路径 |
| External | 发送消息、调用外部 API | 必须人工确认 |
| Irreversible | 删除、资金、法律或医疗相关动作 | 默认禁止或单独强授权 |

## 工具调用前检查

- 当前 Agent 是否被授权？
- 工具是否能访问所需数据分类？
- 输入是否包含外部不可信内容？
- 是否包含 RumorClaim 或其他未验证线索；如包含，是否由用户明确请求 `include_unverified`？
- 调用是否会写入、外发或不可逆？
- 是否需要用户确认？

## 工具调用后处理

- 记录审计事件。
- 记录输出来源。
- 标记产生的派生事实。
- 标记产生的 RumorClaim 为未验证 lead，不能作为事实。
- 对写入和外发动作提供撤销或补救入口。
