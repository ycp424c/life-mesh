# ADR-0002: 静态 Web 项目看板

状态：accepted
日期：2026-06-26

## Context

LifeMesh 需要持续跟踪项目进度、规划和架构。仅靠 Markdown 文档不利于快速扫描当前状态，也不利于在项目演进中发现路线图、风险、文档和决策之间的漂移。

当前项目仍处于文档和规划阶段，尚未进入业务代码实现，也没有选定前端框架或构建链。

## Decision

新增 `dashboard/` 作为静态 Web 项目看板。

看板采用原生 HTML、CSS 和 JavaScript 实现，不引入框架、构建工具或外部依赖。看板状态集中维护在 `dashboard/project-state.js`。

## Consequences

正向影响：

- 项目状态、路线图、架构、风险和决策可以被快速扫描。
- 后续每次项目推进都有明确的状态同步入口。
- 不提前绑定前端技术栈。

代价：

- 需要手动维护看板数据。
- 后续若项目复杂度上升，可能需要迁移到文档生成或前端框架。

## Alternatives Considered

- 只维护 Markdown 文档：不选择，因为缺少可视化总览。
- 直接创建 React/Vite 应用：暂不选择，因为当前没有业务代码和构建链需求。
- 使用外部项目管理工具：暂不选择，因为项目核心状态应保留在仓库内。

## Follow-ups

- 持续更新 `dashboard/project-state.js`。
- 在进入代码实现前评估是否需要自动从 Markdown 提取看板数据。
- 当看板字段稳定后，可补充状态校验脚本。
