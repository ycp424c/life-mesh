# ADR-0001: 文档先行初始化

状态：accepted
日期：2026-06-26

## Context

LifeMesh 的核心问题涉及个人数据、长期记忆、Agent 权限、安全审计和渐进式路线图。过早落代码会把尚未确定的数据边界和治理规则固化进实现。

## Decision

先初始化纯文档结构，不写业务代码、不绑定技术栈、不接入真实个人数据。

文档结构按愿景、治理、领域、架构、路线图、决策记录、研究材料和模板分区。

## Consequences

正向影响：

- 后续实现有稳定的产品和治理基线。
- 数据、安全和 Agent 权限问题不会被实现细节掩盖。
- 可以通过 ADR 持续记录关键决策。

代价：

- 当前阶段没有可运行软件。
- 需要继续补充用户故事、数据源接入评估和最小可行架构。

## Alternatives Considered

- 直接创建应用脚手架：暂不选择，因为技术栈和最小产品边界还未确定。
- 只写单个 README：暂不选择，因为本项目需要长期维护治理、数据模型和安全决策。

## Follow-ups

- 补齐第一批数据源接入评估。
- 定义第一版用户故事。
- 决定第 1 阶段静态知识数字化的最小可行范围。
