# Principles

状态：draft
最后更新：2026-06-26
职责边界：记录跨产品、数据、架构和实现都应遵守的长期原则。

## 原则

1. 数据归用户。
2. 原始数据先进入 Raw Vault，不直接进入 Agent 上下文。
3. 所有派生事实都要能追溯来源。
4. 权限按任务、数据类型、时间范围和动作风险分级。
5. 长期记忆必须支持查看、修改、删除和过期。
6. 推断记忆不能伪装成确定事实。
7. 高敏感数据默认后置接入。
8. 自动执行必须比人工建议拥有更严格的确认和审计。
9. 每个 Agent 能力都应有独立权限、日志和限流。
10. 任何便利性都不能绕过用户控制权。

## 设计含义

- 系统需要同时支持文档库、结构化事实、向量索引、知识图谱和事件时间线。
- Agent 接口层应以工具和资源暴露能力，而不是暴露整个数据库。
- 审计日志是核心产品能力，不是运维附属品。
- 撤销、删除和纠错流程需要从第一天进入设计。
