# AGENTS.md

本项目默认使用中文沟通。

## 当前项目阶段

LifeMesh 目前处于文档与项目看板初始化阶段。除非用户明确要求，否则不要新增业务代码、脚手架代码、依赖配置或运行时目录。

允许修改的内容：

- Markdown 文档
- 决策记录
- 研究笔记
- 文档模板
- 项目说明类文件
- `dashboard/` 下的静态 Web 看板文件

## 工作方式

- 修改前先查看当前目录结构和已有文档。
- 保持文档结构可演进，不把临时想法直接写成不可变结论。
- 对产品、架构、安全、数据模型、权限边界的关键选择，优先写入 ADR。
- 新增数据源、Agent 能力或自动化动作前，先更新对应治理文档。
- 不要把敏感个人数据样例写进仓库；需要示例时使用脱敏、虚构数据。

## 看板与文档同步规则

- 长期事实以 `docs/**` 和 `docs/05-decisions/` 为准；`dashboard/` 是可视化入口，不得绕过正式文档或 ADR 直接形成项目结论。
- 任何会改变项目进度、路线图、架构、治理、安全、风险、决策或当前工作重点的修改，都必须同步检查并更新 `dashboard/project-state.js`。
- 如果看板展示结构本身需要变化，必须同步更新 `dashboard/index.html`、`dashboard/styles.css` 或 `dashboard/app.js`，并在 `docs/08-dashboard/project-board.md` 记录维护规则变化。
- 如果新增、重命名或归档长期文档，必须同步更新 `docs/README.md` 和看板中的文档健康区。
- 如果新增重大产品、架构、数据、安全或 Agent 能力决策，必须新增 ADR，并同步更新看板的决策区。
- 如果修改路线图阶段、阶段状态或验收标准，必须同步更新 `docs/04-roadmap/` 相关文档和看板路线图区。
- 如果修改架构层、数据流或 Agent 接入边界，必须同步更新 `docs/03-architecture/` 相关文档和看板架构区。
- 如果修改风险、权限、删除、审计或威胁模型，必须同步更新 `docs/01-governance/`、`docs/07-security/` 相关文档和看板风险区。
- 每次交付前必须检查 `README.md`、`docs/README.md`、`docs/04-roadmap/`、`docs/03-architecture/`、`docs/05-decisions/` 与看板是否一致。
- 每次交付前必须说明看板是否已同步；若未改看板，需要说明为什么看板状态不受影响。只更新文档或只更新看板时，任务不能视为完成，除非明确说明另一侧无需更新的原因。

## 文档约定

- 每份长期文档应包含：状态、最后更新日期、职责边界。
- `docs/05-decisions/` 存放已经做出的重要决策。
- `docs/templates/` 存放可复制的文档模板。
- `docs/06-research/` 存放外部材料、对话摘要和调研结论。
- `docs/08-dashboard/` 存放项目看板的维护说明和同步约束。

## 质量要求

- 不引入无来源的技术结论；如果是推断，需要明确标注。
- 不把“愿景”“需求”“架构”“实现计划”混在同一份文档里。
- 任何涉及隐私、安全、授权、审计的设计都要写清风险和撤销机制。
- 不允许让 `dashboard/` 展示过期状态。看板无法确定时，应在看板数据中标记为 `Needs review`，而不是保留旧结论。
